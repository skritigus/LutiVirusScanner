## Введение

Данный документ описывает план тестирования десктопного приложения LutiVirusScanner, предназначенного для проверки файлов и папок на наличие вредоносного ПО через сервис VirusTotal API.
Цель тестирования — определить корректность работы функционала и соответствие требованиям, указанным в документе SRS.md

<a name="items"/>

## Объект тестирования

Объектами тестирования являются функциональные возможности приложения:
* Проверка выбранных файлов на вирусы через VirusTotal API;
* Проверка папок с учётом фильтров;
* Просмотр результатов проверки;
* Настройка фильтрации проверяемых файлов;
* Обработка ошибок API, отсутствия соединения или неправильного API-ключа;

<a name="quality"/>

## Атрибуты качества

1. Функциональность:
    - Полная реализация всех заявленных функций проверки, фильтрации и отображения результатов; 
    - Корректность отправки данных в VirusTotal API и получения ответов.
2. Удобство использования:
    - Соответствие интерфейса мокапам;
    - Интуитивно-понятные элементы UI.
3. Безопасность:
    - Защищённое хранение API-ключа.
	- Исключение передачи отчётов сторонним сервисам.
4. Доступность:
    - Корректная работа на поддерживаемых ОС: Windows и Linux.

<a name="risk"/>

## Риски

К рискам можно отнести:
* Некорректная работа при отсутствии подключения к Интернету;
* Блокировка API-ключа при превышении лимитов VirusTotal;
* Непредвиденные ошибки при обращении к API.

<a name="features"/>

## Аспекты тестирования

В ходе тестирования планируется проверить реализацию основных функций приложения, описанных в SRS:
* Проверка отдельного файла
* Проверка папки
* Просмотр результатов проверки
* Настройка фильтров проверки

<a name="feature_1"/>

### Проверка отдельного файла
Этот аспект необходимо протестировать на:
* Выбор файла через проводник;
* Успешная отправка файла в VirusTotal API;
* Обработка статуса проверки (clean / malware);
* Ошибки сети;
* Ошибка при неверном API-ключе;

<a name="feature_2"/>

### Проверка папки
Этот аспект необходимо протестировать на:
* Корректное сканирование всех файлов, подходящих под фильтры;
* Поведение при пустой папке;
* Поведение при ошибке сети;
* Пропуск файлов, запрещённых фильтрами.

<a name="feature_3"/>

### Просмотр результатов проверки
Этот аспект необходимо протестировать на:
* Отображение краткого отчёта: количество обнаруживших угрозу движков;
* Корректность статуса файла;
* Открытие окна детального отчёта;
* Соответствие отображения мокапам;
* Корректная обработка данных с VirusTotal API.

<a name="feature_4"/>

### Настройка фильтров проверки
Этот аспект необходимо протестировать на:
* Включение/исключение расширений (.exe, .txt и т.д.);
* Исключение указанных имён файлов и папок;
* Корректность фильтрации при сканировании папки;

<a name="feature_5"/>

### Ввод и проверка API-ключа
Этот аспект необходимо протестировать на:
* Ввод API-ключа при первом запуске приложения;
* Проверка корректности ключа через тестовый запрос к VirusTotal API;
* Отображение ошибок при неверном ключе;
* Возможность изменения API-ключа через настройки;
* Сохранение нового ключа в шифрованном формате для последующих запусков приложенияж

<a name="nonfunctional_requierements"/>

## Нефункциональные требования:
* Соответствие интерфейса мокапам;
* Полностью функциональная работа на разных Windows и Linux;
* Интуитивная понятность интерфейса.
